## 本次完整 Review 新发现并修复

- 每周上限原先只由求解器稳定执行，现统一覆盖拖拽、手工/加班、公开替班报名和换班终审。
- 禁止搭档校验的排除参数误把人员 ID 当作分配 ID，现按分配 ID 正确排除原换班人。
- 公开替班终审原先需要粘贴 UUID，且可批准未报名人员；现返回实名候选列表并只能选择有效报名者。
- 草稿重生成会删除手工岗位；现保留手工岗位和已有手工分配。
- 任务加入计划、编辑、取消和执行状态可能与 `DutySlot`/`Assignment` 脱节；现已建立状态守卫和级联同步。
- 批准请假后原人员仍可能出现在周表或个人排班；现空缺对外不再显示原人员。
- 手工完成/未到岗可重复或对无效分配操作；现只允许有效且待执行的分配，并写完成审计、同步任务终态。
- 月度和分场地统计在删除最后一条分配后可能保留旧值；现重算会清零或删除未锁定的陈旧汇总。
- PDF 解析预览原先只读；现可修改星期、节次、周次、场地，并增删课程时段。
- admin 代传人员列表原先只含“已有课表的人”，无法首次代传；现改用完整启用人员列表。
- 后端应用代码中 12 项 Ruff 静态错误已清理。
