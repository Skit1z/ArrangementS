## 三、分项验收（按 YAML 顶层章节）

### venues（场地）
- 三场地初始化：黄楼（HL，固定班次）、蓝厅（LT，按任务）、图书馆报告厅（TSG，按任务）— `init_data.py`
- 黄楼 6 班次：每班 `credited_minutes=120`（2.0h 固定）；工作日 2 人 / 周末 1 人 / 调休工作日 2 人（由 `day_rule_service.resolve_required_people` + `SpecialDate.day_type=workday` 实现）— `test_init_data_seeds_venues_and_multipliers`、`test_weekday_two_people` 等
- 事件场地：默认 2 人、最少 1 人、提前 30 / 收尾 30、同场地完整值班时间重叠禁止 — `task_service._validate_and_check_overlap`，`test_same_venue_overlap_rejected`

### hours（工时）
- 30 分钟为单位，按"加权汇总后整体向上取整" — `hours.py` + `test_hours.py`
- 默认倍率：00:00–08:00 ×2.0、19:00–24:00 ×2.0，priority=10 — `init_data.ensure_multiplier_rules`
- 重叠规则：最高优先级生效、不重复乘 — `multiplier_service`
- 双指标：`balance_minutes`（月度公平）、`completed_minutes`（实际统计） — `schedule_stats`
- 未到岗：completed=0、balance 保留、不影响后续排班 — 见 blocker #4

### semester（学期）
- 固定 20 周（`create_semester` 校验）；admin 录 `first_monday` — `semester_service`
- 课程缓冲：可选，默认开启时 +10 分钟 — `generate_intervals(buffer_minutes=10)`
- 6 个节次组时间映射、教学楼前缀（B→main、02-→second）— `CoursePeriodRule` / `BuildingCodeRule` + `test_resolve_period_main_vs_second`

### admin_bootstrap
- `admin` / `admin1234` 幂等种子；可创建其他 admin；不能停用自己 — `init_data.ensure_default_admin`、`admins` 路由

### vacation（假期）
- admin 手动创建；学期结束自动失效课表与不可值班区间（见 blocker #9）；可值班白名单手动录入；非白名单不参与假期排班（见 blocker #10）；黄楼假期默认 1 班/天 × 1 人 — `vacation_service` + `slots._templates_for_day`

### holidays（节假日）
- 数据源 holiday-cn；admin 确认前不生效；支持手动录入兜底 — 见 blocker #11

### account_import（人员导入）
- 必填列：学号/班级/姓名/手机号/困难等级；可选：身份证号/银行卡号；用户名=学号；初始密码=学号+姓名拼音首字母（小写） — `people_import.py` + `test_people_import.py`
- 不强制首次改密（`force_password_change: false`）

### scheduling（排班）
- 仅排入选人员池；每周生成；admin 手动发布；月度公平目标；OR-Tools CP-SAT；单线程可复现 — `solver.py:168-169`
- 10 项硬约束全部建模（时间不重叠、课程冲突、审核通过不可值班、暂停排班、每周最多班次、禁止场地、禁止日期时间、禁止同班人员、非池排除、假期白名单）— solver `Add(...)` + `eligibility.py`
- 强制安排：仅允许非重叠硬约束、需理由、时间重叠永不允许 — `draft_service` forced 语义 + `schedule.test.ts` forced_reason

### workflow（业务流）
- 请假 admin 审核；换班 targeted+open 两模式 + admin 终审；班次结束自动完成（见 blocker #9 配套 `auto_complete_ended`）；admin 可标记未到岗 — `leave_service` / `swap_service` / `execution_service` + 前序用户移动端端到端验证

### statistics（统计）
- 分场地动态存储、无硬编码场地列（见 blocker #12）；月度重算与锁定、工时调整、Excel 导出（敏感字段不入表） — `schedule_stats` / `stats_export` + admin 统计页（阶段六(5/n)）

## 六、已知限制（如实标注）

1. **`FIELD_ENCRYPTION_KEY` 无自动轮换迁移**：换密钥会导致历史密文不可解；轮换需手写一次性脚本（解密→重加密→原子切换），操作手册已说明。
2. **bootstrap admin 无强制改密 DB 字段**：依赖程序性约束（手册与 README 提示首登即改）；如需硬约束，需加 `password_changed_at` 列 + 登录重定向。
3. **浏览器端到端未在本环境复跑**：需本机 Postgres；前序会话已跑通一次完整闭环（提交请假→落库→admin 可见）。三处新接口由 HTTP 级 pytest 等价验证。
4. **请假/换班无自动过期**：YAML 无对应 blocker；`expired` enum 值当前由撤回/admin 拒绝覆盖生命周期，未设定时清理。

以上限制均不影响验收清单红线项的达成。
